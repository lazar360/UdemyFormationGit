Udemy - Formation Git et GitHub : de Débutant à Confirmé

Pour configurer git

  • git config --global user.name "Nicolas GAUTIER"
  • git config --global user.email monemail.com
  • git config --global --list
  • git init

Les commandes pour gérer :

l'indexation

  • git add [nom du fichier]
  • git reset [nom du fichier]
  • git add .

le commit

  • git commit -m"écrire un message"

Afficher les modifications en cours

  • git diff [nom de fichier]
  • git diff
  • git diff --cached [nom de fichier]
  • git diff --cached

Voyager dans l'historique

Afficher la liste des commits

  • git log
  • git log -n
  • git show [numéro SHA1 (= du commit) ou tag] se mettre sur le numéro double cliquer puis clic de mollette

Se déplacer dans les commits

  • git check out [numéro du commit ou tag]
  • git checkout master

Créer des tags pour se déplacer plus vite dans l'historique

Le tag MASTER se positionne toujours sur le dernier commit. Le tag créé reste attaché au commit sur lequel on l'a créé. Le tag permet de naviguer plus facilement et d'identifier une version.

  1. 1/ Se placer sur le commit : git check out [numéro du commit ou tag]
  2. 2/ Utiliser les commandes :
    • git tag [nom du tag]
    • git tag --delete [nom du tag]
    • git tag [nom du tag] -m"écrire u message comme un commit"
